Splash Page

Technical Description

The splash screen for "Good Health and Well-being" is designed to provide a visually engaging introduction while ensuring readability and accessibility. It features a background image with a blur effect, a centered text container, and an animated logo at the bottom.

Key HTML Elements and Their Purpose:

  1. <html lang="en"> – Defines the document's language as English to improve accessibility for screen readers.
  2. <body> – Contains all visual elements and is styled for full-screen display with centered alignment.
  3. – A fixed-position background that fills the viewport, ensuring a dynamic and immersive visual effect.
  4. – A semi-transparent text container positioned above the background, ensuring high contrast for readability.
  5. <iv class="animated-logo">– A logo with a heartbeat animation, providing an engaging transition effect for users.

Accessibility

This splash screen includes several accessibility-focused features to ensure a better user experience for all users, including those with disabilities:

1.Readable and High-Contrast Text

  • The text container has a semi-transparent dark background (rgba(0,0,0,0.4)) to ensure good contrast against the background image.
  • The white font color (color: white;) enhances readability.

2.Language Specification (lang="en")

  • Helps screen readers interpret the content correctly.

3.Avoiding Background Distractions:

  • The blurred background ensures that the text remains the primary focus, reducing visual clutter.

4.Animation Accessibility:

  • >The heartbeat animation is smooth (ease-in-out) and does not use excessive motion, reducing discomfort for users with motion sensitivity

5.Text Hierarchy & Semantic HTML:

  • Headings "h1" provide clear structure for screen readers and improve SEO.
  • Group members are listed as individual

    elements for better readability instead of being in an unordered list

6.Keyboard and Screen Reader Support:

  • The page does not rely on hover effects or complex interactions, ensuring it remains usable for users with keyboard-only navigation or screen readers.

Volunteer Page

Technical Description

The HTML and CSS of the page are structured to create a user-friendly, aesthetically pleasing, and responsive design focused on volunteering opportunities.

1.HTML Structure & Key Tags :

  • The layout is organized using semantic HTML elements like <header>, <nav>, <main>, <section>, and <footer>. This enhances readability and SEO.
  • <nav> contains a navigation bar with links to different pages (Home, Tips, Volunteer, etc.). The inclusion of a profile icon using <i> tags adds a personal touch for users with accounts.
  • <div> elements are used generously to structure content blocks, such as cards, reviews, and the footer
  • The volunteer programs are displayed using <div class="card"> elements, each containing details about a specific program. Each card has icons represented by <i> tags for visual representation of activities.
  • <table> tags are used to present program fees and duration in an organized manner.
  • <textarea> and <button> tags in the review section allow users to submit feedback, enhancing interactivity.

2.CSS Styling & Key Properties:

  • The layout is enhanced by using CSS Flexbox (display: flex;) and Grid properties for structured alignment of elements
  • Tailwind CSS is used for styling the buttons and cards, providing a clean and modern aesthetic. For example, classes like btn, btn-view, and btn-apply are applied to style buttons for different functionalities.
  • Icons from FontAwesome are used to convey meaning quickly through visual representation, enhancing user experience.
  • The dropdown menu for sorting programs uses simple CSS for visibility and hover effects
  • The .container class is used for consistent padding and alignment across sections.

3.Styling Choices & Purpose:

  • Cards Layout: Divides each program into easily readable sections with images, text, and buttons for clarity
  • Consistent Spacing: Padding and margin are applied for a well-organized structure, ensuring elements are visually distinct and not cluttered
  • Visual Hierarchy: Headings, buttons, and ratings are made more prominent with larger font sizes and contrasting colors.

Accessibility

Accessibility has been considered by ensuring the following aspects are addressed:

1.Semantic HTML:

  • Proper use of <header>, <nav>, <section>, <footer> improves accessibility by making content easier to navigate for screen readers.
  • <a> tags are used for navigation instead of JavaScript-based links, allowing better compatibility with screen readers and keyboard navigation.

2.ARIA Attributes:

  • <button> elements are used instead of clickable <div> or <span>, ensuring accessibility for keyboard users and screen readers.
  • The use of aria-label where appropriate can enhance description of icons and interactive elements (though currently missing, it's recommended to add them)

Content Page

Technical Description

The 'Healthy Foods' page of the HealthWell website is implemented using HTML and CSS to create a structured, accessible, and user-friendly interface. The layout is organized with key components like the header, sidebar, content sections, and footer, ensuring a clear navigation experience.

HTML Structure

1.Header (<header>)

  • Contains the website logo and navigation menu wrapped in a <nav> tag.
  • Uses an unordered list (<ul>) to provide links to different pages, ensuring semantic structure.
  • Includes a profile icon linked via a <li> tag with the class profile-icon for better accessibility and recognition.

2.Sidebar (<div class='sidebar'>)

  • Provides quick links to different sections of the page via <a> tags for easier navigation.
  • Fixed positioning ensures it remains visible during scrolling, improving user experience.

    • 3.Content Container (<div class='container'>)

    • Contains all main content sections organized under individual <div> blocks with the class section.
    • Each section provides information about healthy foods, benefits, and dietary guidelines.
    • Uses images (<img>) to visually enhance the content. The <img> tags include alt attributes for accessibility.

    CSS Styling

    1.Layout and Structure.

    • Flexbox is used within the header and footer for alignment and spacing (display: flex; justify-content: space-between;).
    • The sidebar is fixed using position: fixed; for consistent visibility.

    2.Styling Choices

    • The overall color scheme uses various shades of green (#27ae60, #254222) to convey a natural, health-focused aesthetic.
    • Cards (<div class='section'>) are styled with padding, margin, rounded corners, and subtle shadowing for enhanced readability and visual appeal.

    Accessibility

    1.Semantic HTML

    • The use of <header>, <nav>, <footer>, <ul>, <li>, and <main> elements contributes to a well-structured, accessible layout.

    2.Image Accessibility

    • All <img> tags have appropriate alt attributes describing the images, enhancing accessibility for screen readers.

    3.Keyboard Navigation & Focus Management

    • Links and buttons are styled with clear focus states (:hover effects) to improve usability for keyboard users.

    4.Color Contrast

    • Text and background colors are chosen to ensure sufficient contrast, meeting accessibility guidelines.

    Challenges and Lessons Learned

    1.Layout Structure and Responsiveness:

    • Initially, positioning the sidebar and making the page content responsive were difficult. This was resolved by applying position: fixed; for the sidebar and using appropriate width and margin-left properties for the content.

    2.Overlapping Elements:

    • The fixed header overlapped with the page content. Adding margin-top to the main content container fixed this issue.

    #.Image Sizing Issues:

    • Images were not displaying properly in smaller viewports. Using width: 80%; and max-width: 800px; within sections resolved this problem.
